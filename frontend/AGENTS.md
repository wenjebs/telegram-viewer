# Frontend Agent Rules

## Package Manager

- Use `bun` as the package manager (not npm/pnpm/yarn)
- Run scripts with `bun run <script>`

## Code Style

- Format with oxfmt (config: `.oxfmtrc.json`): no semicolons, single quotes, trailing commas, 80 char line width
- Lint with oxlint (config: `.oxlintrc.json`): react, typescript, unicorn, import plugins enabled
- Run `bun run check` before committing to auto-fix formatting and lint issues
- Do not add `import React` — React 19 JSX transform handles this

## Stack

- **Framework**: TanStack Start (React 19, SSR)
- **Router**: TanStack Router (file-based routing in `src/routes/`)
- **Data Fetching**: TanStack Query (`useQuery`, `useInfiniteQuery` for cursor pagination)
- **Styling**: Tailwind CSS v4
- **Testing**: Vitest + React Testing Library
- **Type Checking**: tsgo (`@typescript/native-preview`) — preferred over tsc for speed
- **Build**: Vite 7

## Conventions

- Use `#/*` import alias for `./src/*` paths (configured in package.json `imports`)
- Route files live in `src/routes/` and are auto-generated by TanStack Router
- Use `createFileRoute` / `createRootRoute` for route definitions
- Use `createServerFn` for server-side logic
- Store navigable/filterable UI state in URL search params via `validateSearch` + Zod schema on the route, read with `useSearchParams()` hook. Keep ephemeral state (loading, drag, modals) in React state.
